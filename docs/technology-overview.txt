Generated from the current repository state on 2026-04-15.

1. Project Summary

Atlas is a Next.js + React + TypeScript demo application for adaptive hotel review capture. It combines:

- a frontend review flow
- local demo-state persistence
- an offline CSV ingestion pipeline
- heuristic gap scoring over hotel-review data
- OpenAI-powered follow-up generation, review enhancement, title generation, embeddings, and optional photo labeling

The app is built as a demo product rather than a production SaaS stack. It does not currently use a traditional database or authentication layer. Instead, it uses generated JSON artifacts plus a local JSON demo store.

4. Application Architecture

4.1 Offline phase

The offline phase is driven by `npm run ingest`.

Flow:
- load `.env.local`
- parse raw CSVs
- normalize property and review records
- compute candidate gaps
- build evidence snippets and rationales
- build RAG chunks
- create embeddings and cache them
- write generated artifacts into `data/generated`

Key files:
- `scripts/ingest-data.mjs`
- `scripts/lib/parse.mjs`
- `scripts/lib/catalog-builder.mjs`
- `scripts/lib/attribute-catalog.mjs`
- `scripts/lib/text.mjs`

4.2 Runtime phase

The runtime app loads the generated JSON catalog and serves a guided review experience.

Flow:
- load seeded demo customers
- hydrate stays with property data from the generated catalog
- let the user review a stay
- use heuristics + optional OpenAI/RAG to determine follow-up questions
- let the user answer and preview structured updates
- enhance the review with grammar fixes and a generated title
- persist demo progress in the local JSON store

Key files:
- `src/app/page.tsx`
- `src/components/reviewiq/reviewiq-demo-shell.tsx`
- `src/components/reviewiq/reviewiq-client.tsx`
- `src/components/reviewiq/stampedia-journal.tsx`
- `src/lib/reviewiq/runtime-data.ts`
- `src/lib/reviewiq/demo-store.ts`
- `src/lib/reviewiq/gap-coverage.ts`
- `src/lib/reviewiq/session.ts`

5. API Surface

5.1 Demo-data API routes

- `POST /api/demo/reset`
  Functionality:
  - resets the seeded demo customer profiles
  - clears local demo review state

- `POST /api/demo/review`
  Functionality:
  - saves a completed review
  - persists title, polished text, answer previews, and uploaded photo data
  - marks the stay as reviewed

- `POST /api/demo/trip`
  Functionality:
  - renames a trip inside Stampedia

5.2 Review intelligence API routes

- `POST /api/session/start`
  Functionality:
  - starts the adaptive follow-up session
  - returns up to two follow-up questions

- `POST /api/session/answer`
  Functionality:
  - previews a structured candidate fact update from a user answer

- `POST /api/session/polish`
  Functionality:
  - enhances review grammar/spelling/punctuation
  - generates a short review title

- `POST /api/photos/classify`
  Functionality:
  - optionally labels uploaded hotel photos with concise hospitality labels

7. Core Logic Systems

7.1 Gap scoring engine

Implemented in:
- `src/lib/reviewiq/gap-coverage.ts`
- `scripts/lib/catalog-builder.mjs`

Functionality:
- scores property attributes by importance
- checks whether the draft already covers them
- identifies stale, contradictory, or weakly covered facts
- determines which questions are still worth asking

7.2 Custom RAG system

Implemented in:
- `scripts/ingest-data.mjs`
- `src/lib/reviewiq/session.ts`

Functionality:
- embeds offline property/review chunks
- embeds runtime gap queries
- retrieves semantically relevant evidence
- uses that evidence to ground the adaptive follow-up question

7.3 Demo-store hydration

Implemented in:
- `src/lib/reviewiq/demo-store.ts`
- `src/lib/reviewiq/runtime-data.ts`

Functionality:
- loads generated property summaries
- hydrates seeded stays with current property metadata
- updates stale-focus labels from the latest catalog data

8. Important Product Constraints

- No production database yet
- No authentication or account system yet
- Demo persistence is file-based, not multi-user safe
- OpenAI enhancer currently expects an API key
- Storybook/Vitest/Playwright tooling is installed and configured, but the current repo is primarily demo-app code rather than a large test suite

9. Recommended One-Line Description

Atlas is a Next.js demo product that ingests hotel and review CSVs, computes stale or missing property facts, guides travelers through an adaptive review flow, and uses OpenAI plus local heuristics to turn fresh traveler feedback into structured, review-driven intelligence.
